# DTF Radar

DTF Radar is a lightweight competitor intelligence MVP for the DTF market.

It helps sellers, brands, and distributors monitor:
- pricing changes
- product launches
- promotional updates
- messaging shifts
- competitor activity trends

## Current status
This repository is no longer only a front-end prototype. It now includes an early backend-ready MVP skeleton with:
- Prisma data model
- competitor create/list/detail flows
- discovery suggestion + tracked page persistence
- alerts / change events data layer
- dashboard summary aggregation
- seed / demo data support

## Local development

### 1. Install dependencies
```bash
npm install
```

### 2. Create environment file
Copy `.env.example` to `.env` or `.env.local`:

```bash
cp .env.example .env
```

Then update `DATABASE_URL`.

Example:
```bash
DATABASE_URL="postgresql://USER:PASSWORD@HOST:5432/dtf_radar?schema=public"
```

### 3. Generate Prisma client
```bash
npx prisma generate
```

### 4. Push schema to database
For the current MVP stage, the fastest setup path is:

```bash
npx prisma db push
```

### 5. Seed demo data
```bash
npm run db:seed
```

### 6. Start dev server
```bash
npm run dev
```

Open:
```bash
http://localhost:3000
```

## Recommended database setup flow

### Fast local MVP flow
Use this while iterating quickly:

```bash
npx prisma generate
npx prisma db push
npm run db:seed
npm run dev
```

### Migration-based team flow
When you want schema history checked into Git:

```bash
npx prisma migrate dev --name init
npm run db:seed
npm run dev
```

If migrations already exist later, new teammates can run:

```bash
npx prisma migrate dev
npm run db:seed
```

## Prisma 7 note
This project uses Prisma 7 config via `prisma.config.ts`.

That means:
- datasource URL is configured in `prisma.config.ts`
- `DATABASE_URL` must be available when running Prisma commands
- if Prisma complains about missing env vars, make sure `.env` is loaded or prefix the command manually

Example:
```bash
DATABASE_URL="postgresql://USER:PASSWORD@HOST:5432/dtf_radar?schema=public" npx prisma generate
```

## Demo data included

- demo user

- demo change events

This is enough to populate:
- competitors list
- competitor detail
- alerts list/detail
- dashboard summary

## Current limitations
- discovery suggestions are rule-derived from `websiteUrl`, not real crawling yet
- change events are not generated by a crawler yet
- page snapshots / diff engine are not implemented yet
- auth / multi-workspace handling still uses a temporary default workspace id
