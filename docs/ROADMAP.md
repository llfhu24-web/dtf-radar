# DTF Radar Roadmap

## Product direction
DTF Radar is aimed at DTF-related sellers and distributors who need a faster way to monitor competitor websites.

## Stage 1 — UX prototype
Completed:
- landing page
- dashboard
- competitors list
- competitors detail
- alerts list
- alerts detail
- add competitor flow
- page discovery review

## Stage 2 — UI system cleanup
Next:
- extract more reusable UI components
- normalize page headers, badges, tables, and stat blocks
- improve responsive behaviors
- add empty states and loading states

## Stage 3 — Functional data model
Planned:
- competitor CRUD
- tracked page storage
- page discovery pipeline
- snapshot persistence
- change event schema
- alert detail enrichment

## Stage 4 — Monitoring engine
Planned:
- scheduled crawl jobs
- diff detection
- importance scoring
- DTF-specific keyword tagging
- daily summary generation

## Stage 5 — Delivery and reporting
Planned:
- email brief delivery
- Telegram summaries
- exportable PDF summaries
- saved filters
- team-facing weekly reports
